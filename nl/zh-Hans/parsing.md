# 了解合同解析
{: #contract_parsing}

{{site.data.keyword.cnc_short}} 会返回已解析的合同，并对每个识别到的元素进行分析。

以下部分描述返回的 JSON 如何提供分析。

## Types
{: #contract_types}

`types` 数组包含多个对象，其中每个对象都包含 `nature` 和 `party` 键，这两个键的值用于共同标识相应元素。下表列出了 `nature` 和 `party` 键的可能值。

`nature` 值列出语句需要的操作类型。

|`nature`|描述|
|:----------------:|-----------------------------------------------------------|
|`定义`|此元素用于添加对术语、关系或类似项的澄清。无需执行任何操作来实现此元素，也不会影响任何当事方。|
|`免责声明`|此元素中的 `party` 没有义务履行此元素指定的条款，但并未禁止履行。|
|`排除条款`|此元素中的 `party` 将不履行此元素指定的条款。|
|`义务`|此元素中的 `party` 需要履行此元素指定的条款。|
|`权利`|此元素中的 `party` 保证接收此元素指定的条款。|
|`建议`|此元素中的 `party` 没有义务履行此元素指定的条款，但强烈建议履行。|

## Parties
{: #contract_parties}

`parties` 数组指定合同中列出的参与者。每个识别到的 `party` 对象会按名称列出识别到的当事方，并将其与用于对 `party` 对象角色分类的 `role` 对象进行匹配。可以为合同返回的 `role` 值包括：

|`role`|描述|
|:----------------:|-----------------------------------------------------------|
|`买方`|负责为合同中所列商品或服务付款的当事方。|
|`最终用户`|将与所提供的商品或服务进行交互的当事方，明确区别于`买方`。|
|`无`|没有识别到此元素的任何当事方。此值始终与值为`定义`的 nature 成对使用。|
|`供应商`|负责提供合同中所列商品或服务的当事方。|

## Categories
{: #contract_categories}

`categories` 数组定义语句的主题。目前支持的类别包括：

|`categories`|描述|
|:----------------:|-----------------------------------------------------------|
|`修订条款`|这些元素用于指定在签署合同后对合同的更改或对标准合同进行的变更。用于指合同变更以及协议更改的元素也属于此类别。|
|`资产使用`|这些元素用于指以下情境：双方在依据协议（包括协议中的许可权和限制）履行职责时，一方必须使用另一方的资产（许可证或设备）。|
|`转让`|这些元素用于描述将权利和/或义务转让给第三方。|
|`审计`|此类别包含以下元素：用于指记帐和用于检查另一方帐簿或记录的权利，以及用于指一方检查或审查合规性的权利，或者一方可供执行检查或合规性审查的需求。|
|`业务连续性`|一个狭义类别，其元素用于指销售、合并或其他重大更改对协议另一方的影响。|
|`通信`|这些元素用于指通知或提供声明的需求、关于通信方法的详细信息（交换信息的行动或过程）、联系人代表和联系人信息。此外，这些元素还用于指各方之间交换信息的可接受的方法。|
|`机密性`|此类别包含以下元素：用于描述当事方可以如何使用在完成合同的过程中所了解到的信息；用于讨论维护商业秘密；以及用于描述不披露业务信息。|
|`可交付内容`|此类别包含以下元素：用于指定一方给予另一方商品以换取资金，以及用于描述准备可交付内容。|
|`交付`|这些元素用于指定将`可交付内容`类别中的商品从一方转移到另一方的方法。例如，如果一方销售访问权或云服务，那么获取该访问权的方法就属于此类别。|
|`争议解决`|此类别包含以下元素：用于讨论如何解决缔约各方之间产生的任何争议的规定；例如，通过定义的过程（如仲裁小组）来解决。争议可由除仲裁之外的各种方法进行解决；例如，不可弥补的损害、获得禁令或大意为“您不可就此提起集体诉讼”的声明。争议示例包括人工争议以及有关发票或帐单的争议。此类别还包含用于指定合同适用法律或法律选择（例如，特定国家/地区或管辖区域）的元素。|
|`不可抗力`|这些元素用于指超出当事方控制的破坏性事件。这是特定的责任限制。|
|`赔偿`|这些元素用于指定某些责任的补救措施。赔偿是指合同一方有责任根据另一方的责任对其进行赔偿。赔偿这一事实意味着已经发生了赔偿责任。|
|`保险`|这些元素用于指一方向另一方提供的任何类型的保险责任范围或承保条款。|
|`知识产权`|示例倾向于分为两类：使用专利、版权和商标的正式定义（通常源自法令）；使用发明、著作权和专用技术的更广义概念。此外，还可以针对知识产权提供定义，例如侵犯此类产权时的起诉和索赔权利。`知识产权`包括专利、申请专利的权利、商标、商号、服务标记、域名、版权以及这类权利在全球范围的所有申请和注册，另外还包括图表、工业模型、发明、专用技术、商业秘密、计算机软件程序和其他无形专有信息。对于“第三方代码”，如果代码的任何部分属于其他人，那么需要声明知识产权。|
|`责任`|这些元素描述确定过错何时以及如何与任一方联系在一起的方法。|
|`支付条款和帐单`|一个广义类别，包含用于描述当事方如何付款或收款的元素，其中包括以下元素：用于指合同规定的最终付款方式或一方将如何付款的元素；详细说明一方如何及何时收款，以及各方付款购买或记帐的商品类型的元素；用于指任何种类的费用支付的元素，但用于指特定价格或数字的元素除外；以及用于指合同规定的收款方的元素。|
|`定价和税款`|此类别包含的一些元素用于描述当事方付款或收款所针对的事项。此类别包含的另一些元素用于指与交换的各个可交付内容关联的税款和特定金额（其中包括有关特定元素的成本的详细信息），以及用于描述特定数字或价格计算方法。|
|`隐私`|这些元素用于指定个人信息处理；即，有关特定个人的需要保护的私人信息。此类别是更广义的`机密性`类别下非常具体的子集。|
|`职责`|协议的附属任务，只有一个当事方有权监督和控制这些任务。属于此类别的元素类型因合同的性质而有所不同。|
|`安全和保障`|这些元素用于指数据和系统的物理安全或网络安全保护，以及用于指提高工作场所安全性。|
|`工作范围`|此类别定义合同中已存在的内容与合同中不存在的内容。通常，用于告知当事方如何定义特定命令的元素属于此类别。示例包括“工作说明书”的讨论或对适用通信标准的描述。|
|`子合同`|这些元素用于指根据合同雇佣第三方履行某些职责，以及与合同相关及因合同引起的许可权、权利、限制和后果。|
|`期限和终止`|这些元素用于指合同有效期、日程安排和合同终止条款以及终止产生的任何后果（包括终止时或终止后适用的任何义务）。|
|`保证`|这些元素用于特指当事方可以依赖的后台基本假定。违反保证的后果也属于此类别。|

## Attributes
{: #attributes}

`attributes` 数组指定在语句中识别到的任何属性。此数组中的每个对象都包含三个键：`type`（下表中的属性类型）、`text`（适用的文本）和 `attribute`（属性在文档中的起始点和结束点）。目前支持的属性包括：

|`attributes`|描述|
|:----------------:|-----------------------------------------------------------|
|`Location`|地理位置或区域。|
|`DateTime`|日期、时间、日期范围或时间范围。|
|`Currency`|币值和单位。|

# Assurance
{: #assurance}

{{site.data.keyword.cnc_short}} 对它识别到的每个 `type` 或 `category` 元素都给予保证评级。目前 assurance 只有一个值，即 `High`，表示有明显证据证明，列出的分类能够代表内容。

## Provenance
{: #provenance}

`types` 和 `categories` 数组中的每个对象都包含一个 `provenance` 对象。`provenance` 对象有一个或多个 `id` 键。每个 `id` 键都有一个散列值，可以将该散列值发送给 IBM 以提供反馈或接收支持。
